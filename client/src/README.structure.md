# Client Folder Structure (Hybrid)

This document describes the scalable folder structure for the client app.

## Principles

- Feature-first: group code by domain/module instead of by type.
- Thin routes: routes are lightweight and render module screens.
- Co-location: module UI, hooks, types, and API logic live together.
- Shared code: cross-module UI and utilities live in `components/` and `lib/`.

## Structure

```
client/src
  components/          # shared UI (reusable across modules)
  integrations/        # tanstack-query, router, etc.
  lib/                 # shared utilities
  modules/
    finance/
      entity/
        api/
        components/
        hooks/
        screens/
        types/
        index.ts
      fee/             # future module
      transaction/     # future module (if needed again)
  routes/
    finance/
      entity/
        index.tsx      # thin route -> module screen
  routeTree.gen.ts     # generated by TanStack Router
  router.tsx
```

## Module Conventions

- `screens/` are route-level components.
- `components/` are module-only UI components.
- `hooks/` hold module hooks (often wrapping TanStack Query).
- `api/` contains fetchers and query helpers.
- `types/` contains module types/schemas.
- `index.ts` re-exports the module's public surface.

## Adding a New Module

1. Create `modules/<domain>/<feature>/` with the standard subfolders.
2. Add a thin route under `routes/<domain>/<feature>/index.tsx`.
3. Run `bun run build` in `client/` to update `routeTree.gen.ts`.

Route file:

- `routes/finance/entity/index.tsx` renders `modules/finance/entity/screens/EntityScreen.tsx`.
